Общие входные параметры

POST/GET https://split.daichicloud.ru/api/open/<версия>/<метод>?<параметры>
Параметр Положение Тип Обязательный Описание
version Path Enum Да Версия API.
Accept Header MIME Да Параметр контента для выходных данных. Указать application/json
Content-Type Header MIME Нет Параметр контента для входных данных. Указать application/json при использовании формы json.
Authorization Header String Да(*) Авторизационный ключ (access_token), получаемый в методе авторизации auth. Обязательный параметр для всех методов кроме Авторизации (auth) и Обновления токена (auth/token/refresh).

HTTP POST, GET запросы можно отправлять как в обычном формате (Content-Type: application/x-www-form-urlencoded || multipart/form-data), так и в виде json. Далее в примерах для удобства просмотра все входные и выходные параметры запросов описываются в json-формате.

Индикатором выполнения запросов служат стандартный http status code:
2xx - успешное выполнение
4xx - ошибки на стороне клиента
5хх - ошибки на стороне сервера

Пример ответа при ошибке

Response:
Status: 422 


{
  "errors": {
      "device": "Нет доступа к устройству"
  },
  "message": "Нет доступа к устройству"
}
